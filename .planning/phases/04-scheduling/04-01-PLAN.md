  <action>
    1. Install node-cron: `cd minipilot/server && npm install node-cron`

    2. Create `server/scheduler.js` as an ES module exporting:
       - `buildCronExpression(schedule)` — maps frequency+params to cron string:
         - daily: `${minute} ${hour} * * *`
         - weekly: `${minute} ${hour} * * ${day_of_week}`
         - monthly: `${minute} ${hour} ${day_of_month} * *`
         - once: returns null (handled separately)
       - `initScheduler(db, helpers)` — receives the db instance and helper functions (aiComplete, buildDataSummary, buildColumnStats, extractThemes, uuidv4). Stores them in module-level variables. Calls `rehydrateSchedules()` to re-register all active schedules from DB.
       - `registerSchedule(schedule)` — creates a node-cron task with `cron.schedule(expr, callback, { timezone: 'Europe/Paris', noOverlap: true, name: schedule.id })` and stores in `activeTasks` Map. For `frequency === 'once'`, calls `executeSchedule(schedule.id)` directly instead.
       - `unregisterSchedule(scheduleId)` — calls `task.stop()` and removes from `activeTasks` Map.
       - `executeSchedule(scheduleId)` — the core run function:
         a. Load schedule from DB, verify status === 'active'
         b. Check end_date — if past, set status='completed', stop task, return
         c. Increment edition: `edition_count + 1`
         d. If `source_file_id` is set, the data is already in clean_data (no re-transform needed for v1.1 — per research recommendation, use current workspace data as-is)
         e. Build suggestion: if `template_report_id`, load report from DB and build `{ title: "${report.title} #${editionNum}", description, type: 'bar', columns: [], kpis: [] }`; if `suggestion` JSON stored, parse it and append edition number to title
         f. Call the generate-report logic (reuse the prompt construction + aiComplete + report INSERT pattern from index.js lines 2470-2549, extracted into a helper `generateReportForSchedule(workspaceId, suggestion)` within scheduler.js)
         g. INSERT into schedule_runs with status='success'
         h. UPDATE schedules: edition_count, last_run_at, status (set 'completed' if frequency==='once')
         i. On error: INSERT schedule_runs with status='error', UPDATE schedule status to 'error'
       - `rehydrateSchedules()` — SELECT all schedules WHERE status='active', check end_date, call registerSchedule for each valid one. Log count.

    3. IMPORTANT node-cron v4 rules (from research):
       - Import: `import cron from 'node-cron'` (default import, ESM)
       - NO `scheduled: false` (removed in v4)
       - NO `task.destroy()` (removed in v3+) — use `task.stop()` + Map.delete()
       - Use `noOverlap: true` to prevent concurrent execution overlap
       - Use `cron.validate(expr)` before registering

    4. The module does NOT import express or define routes — it exports pure functions that index.js calls.
  </action>
